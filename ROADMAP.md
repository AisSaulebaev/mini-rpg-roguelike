# Roadmap

Одобрено пользователем 2026-04-20. Порядок сверху вниз — каждый блок закрываем целиком, потом идём дальше. Пункты 5 (свитки) и 10 (рефералы) отклонены.

---

## 1. Новые монстры

5 штук, расширяют `MONSTER_TEMPLATES` в `game.js`:

- **Скелет-лучник** (minDepth 2) — первый удар до сближения (pre-combat shot), низкое HP.
- **Слизь** (minDepth 3) — при смерти делится на 2 мини-слизи с 50% HP родителя. Нужна логика `onDeath → spawn`.
- **Мимик** (minDepth 6) — стоит на карте как 🎁, при открытии сундука с шансом 15% превращается в монстра. Новый тип тайла или флаг на чейсте.
- **Вампир** (minDepth 7) — лечится 50% от нанесённого урона. Новое поле `lifesteal` в шаблоне.
- **Элементал льда** (minDepth 10) — накладывает статус «заморозка» (пропуск хода, переиспользуем `stun` или делаем новый). Высокий крит-резист.

Ассеты: 5 PNG в `img/monsters/` (пайплайн: кириллица→ASCII, как для предметов).

## 2. Элитные монстры

Аффикс-система поверх существующих шаблонов:

```js
const ELITE_AFFIXES = {
  raging:   { name: 'Бешеный',     atkMul: 1.5, hpMul: 1.2 },
  armored:  { name: 'Бронированный', defMul: 2,   hpMul: 1.3 },
  cursed:   { name: 'Проклятый',   bleedChance: 30, hpMul: 1.4 },
};
```

- Шанс спавна: ~8% начиная с depth 5, +1% за этаж, кап 20%.
- Визуально: префикс в имени + подсветка рамки на карте/в бою.
- Дроп: гарантированный эпик (минует `rollItem`, дёргает только `rarity==='epic'`).

## 3. Алтари-события

Новый тайл `altar`, редкий спавн (1 раз каждые 3–5 этажей).

- Массив `ALTAR_EVENTS` с 2–3 вариантами выбора.
- Модалка «Алтарь» с кнопками (как shop-confirm).
- Одноразовые — после активации тайл исчезает.

Примеры:
- «Кровавый алтарь»: +2 ATK навсегда / −5 max HP.
- «Золотой идол»: +100 золота / −50% текущего HP.
- «Осколок судьбы»: +10% крита / −5% уклонения.

## 4. Сеты предметов

- Добавить `setId` в шаблоны в `ITEM_POOL`.
- Бонусы считать в `recalcStats`: группируем экипировку по `setId`, смотрим размер группы, применяем бонус.

Стартовые сеты:
- **Воровской** (3 предм: кинжал, талисман вора, перстень тени) → +15% золота.
- **Воин** (3 предм: меч, шлем, нагрудник) → +2 ATK.
- **Мистик** (2 предм: кольцо, ожерелье) → +5% крита.

UI: пометка сета в тултипе + подсветка когда собран.

## 5. Новые боссы

Сейчас один дракон на depth 10. Добавить:

- **Лич** — depth 20. Раз в 3 хода воскрешает случайного убитого монстра этажа с 50% HP. Отдельный AI-тик.
- **Тёмный рыцарь** — depth 30. DEF 5, но бафнутый `bleedChance` урон его режет (делаем уязвимость через `bleedMul`).

Надо обобщить логику босса — сейчас хардкодед `MONSTER_TEMPLATES.dragon` в `initFloor`. Сделать словарь `BOSS_BY_DEPTH = { 10: 'dragon', 20: 'lich', 30: 'darkknight' }`.

## 6. Ежедневная таверна

В главное меню кнопка «🍺 Таверна».

- MVP клиентский: `meta.lastTavernClaim` (timestamp), при клике если прошло 24ч → +100 душ.
- Защита от читов позже — переносим на D1 (`daily_claims` таблица по uid).

## 7. Ревайв за 10⭐

- Новый пак `revive_hp` в сервере + клиенте, 10⭐.
- На экране смерти кнопка «Воскреснуть за 10⭐».
- При покупке — счёт через существующий `/invoice`, после `paid` восстанавливаем 50% HP, закрываем death-modal, возвращаем на тот же этаж.
- Важно: нельзя скопить ревайвы заранее — только по клику после смерти. Значит в `pending` кредитим поле `revives`, но клиент тратит сразу при смерти.

## 8. Лидерборд

Сервер:
```sql
CREATE TABLE runs (
  uid TEXT,
  depth INTEGER,
  monsters_killed INTEGER,
  souls INTEGER,
  finished_at INTEGER
);
```

- `POST /run` — клиент шлёт на death/win со своей лучшей глубиной. Валидация через `initData`.
- `GET /top?period=all|week` — топ-20.

Клиент:
- Кнопка «🏆 Рейтинг» в главном меню.
- Модалка с таблицей: место, имя (из Telegram user), глубина.

## 9. Мульти-игровой хаб

Архитектурный рефакторинг.

- Перенести игру из корня в `games/rpg/` (index.html, game.js, style.css, img/).
- Корневой `index.html` превращается в лончер: карточки игр (иконка + название + кнопка «Играть»).
- Клик по карточке → `location.href = 'games/rpg/'` (простой способ, без iframe).
- Общая мета (души, Telegram Stars баланс) — решить потом: либо shared localStorage, либо серверная `users` таблица в D1.

Делаем только когда появится ≥2 игр — сейчас смысла мало.
