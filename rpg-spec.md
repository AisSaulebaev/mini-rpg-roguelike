# Мини-RPG Rogue-like — Техническое задание (v2)

## Общая концепция
Пошаговая рогалик-игра на сетке, которая расширяется с глубиной. Игрок двигается по клеткам, в бою с соседним монстром выбирает действие через меню. Каждый этаж — новая комната с монстрами и лутом. Цель — спуститься как можно глубже.

**Циклы игры:**
- **Забег (run):** уровень + снаряжение + HP обнуляются в начале. Копятся в течение забега.
- **Мета (между забегами):** после смерти счёт → мета-валюта → апгрейды базовых статов.

## Технологии
- **HTML + CSS + JavaScript**, без фреймворков и сборки
- Открывается двойным кликом с диска или через Telegram Web App
- Сохранение только мета-прогресса через `localStorage`
- Адаптивно: Telegram Web App (mobile) + десктоп

## Механика

### Ход игрока
Управление: кнопки-стрелки ⬆️⬇️⬅️➡️ (всегда на экране) + клавиатура (стрелки/WASD) + свайпы по сетке.

При нажатии направления:
- **Пустая клетка / предмет / лестница** → игрок двигается, подбирает предмет / спускается
- **Стена поля** → ход не тратится
- **Монстр в соседней клетке** → движения не происходит, открывается меню боя

### Меню боя (монстр рядом)
Три кнопки:
1. **⚔️ Атака** — наносишь урон = `max(ATK - monster.DEF, 1)`. Монстр отвечает.
2. **🛡️ Защита** — ты не атакуешь. Входящий урон снижен вдвое (`floor`). Монстр бьёт со сниженным уроном.
3. **🏃 Уйти** — двигаешься в выбранную сторону (если свободно), **монстр бьёт в спину полным уроном**.

*Примечание:* меню боя появляется **вместо** хода движения. То есть когда рядом монстр, тыкая в его направление — получаешь меню. В другие стороны (если там нет монстра) — ходишь нормально.

### Ход монстров
После каждого действия игрока (атака / защита / уход / движение):
- Если игрок рядом (4-связность) — монстр **уже отработал** в меню боя, пропускает ход
- Иначе — идёт к игроку на 1 клетку (greedy по max|dx|,|dy|)
- Два монстра не могут стоять на одной клетке. Если путь занят — пытается другую ось. Если и там занято — стоит.

### Характеристики игрока
- **HP** / **maxHP** — здоровье
- **ATK** — атака (базовая + от снаряжения + от уровня + от меты)
- **DEF** — защита (аналогично)
- **Level** — уровень (1+)
- **XP** — опыт до следующего уровня
- **Gold** — внутриигровое золото (тратится ни на что, только считается в счёт)
- **Depth** — текущий этаж

### Монстры
Базовые статы, множитель глубины = `1 + (depth - 1) * 0.1`:

| Эмодзи | Имя     | HP | ATK | DEF | XP | Gold | Появление       |
|--------|---------|----|-----|-----|----|----|-----------------|
| 👹     | Гоблин  | 8  | 3   | 0   | 3  | 2-5  | этаж 1+         |
| 🧟     | Зомби   | 15 | 4   | 1   | 6  | 4-8  | этаж 3+         |
| 👻     | Призрак | 10 | 6   | 2   | 10 | 6-12 | этаж 5+         |
| 🐉     | Дракон  | 40 | 8   | 3   | 30 | 30-50| каждый 10-й этаж|

На боссовых этажах (10, 20, 30…) — только босс + возможно 1 слабый моб.

### Система уровней
- XP даётся за убийство монстра
- Порог уровня: `level_xp(n) = 10 * n * (n + 1) / 2` → 10, 30, 60, 100, 150…
- При level up:
  - +3 maxHP (и текущее HP тоже +3)
  - +1 ATK каждые 2 уровня (lvl 2, 4, 6…)
  - +1 DEF каждые 3 уровня (lvl 3, 6, 9…)
  - Всплывашка "LEVEL UP!"

## Предметы и снаряжение

### Подбираемые на поле
- **🪙 Монета** — +1-3 gold
- **🧪 Зелье** — добавляется в инвентарь (не используется сразу). До 3 штук.
- **🎁 Сундук** — открывается при подборе, выдаёт:
  - 40% — снаряжение случайной редкости (см. ниже)
  - 30% — 10-30 gold

## Мета-прогрессия

### Валюта
**Души** 💀 — мета-валюта, начисляется после смерти.

Формула счёта:
```
score = depth * 20
      + monstersKilled * 5
      + chestsOpened * 10
      + bossesKilled * 100
      + level * 15
      + goldCollected (как есть)
```

**Души = floor(score / 10)**

### Апгрейды в магазине (перед забегом)
| Апгрейд            | База | Рост цены |
|--------------------|------|-----------|
| +2 maxHP           | 20 💀| +50%      |
| +1 ATK             | 30 💀| +50%      |
| +1 DEF             | 30 💀| +50%      |
| Стартовое зелье    | 40 💀| +100%     |
| +10% XP навсегда   | 60 💀| +100% (макс 3)|
| +10% gold навсегда | 60 💀| +100% (макс 3)|
| Стартовый меч      | 80 💀| разовая   |

Цена = `base * 1.5^покупок`. Округление вверх.

### Что сохраняется (localStorage)
```js
meta: {
  souls: 120,
  upgrades: { hp: 3, atk: 2, def: 1, startPotion: 1, xpBonus: 0, goldBonus: 0, startWeapon: false },
  stats: { bestDepth: 12, totalRuns: 8, bossKills: 2, totalMonstersKilled: 140 }
}
```

### Что НЕ сохраняется
Текущий забег: HP, уровень, снаряжение, этаж. Всё заново.

## Основной цикл хода
1. Игрок выбирает направление
2. Проверка клетки → ветвление:
   - Стена → ничего
   - Монстр → открыть combat menu
   - Предмет/пусто/лестница → двинуться, обработать
3. Если combat — ждать выбор, применить, проверить HP монстра (убит → XP, gold, убрать)
4. Ход монстров (все, кроме того, что в combat если жив)
5. Проверить HP игрока (0 → экран смерти, начислить души)
6. Перерисовать UI

- Level up — крупный текст "LEVEL UP!" в центре экрана, 1.5с
- Переход на этаж — fade out/in сетки, 300мс

## Управление
- **Десктоп:** стрелки / WASD, цифры 1/2/3 в бою
- **Мобильный:** кнопки-стрелки + свайп по сетке
- **Зелье:** кнопка в инвентаре / клавиша Q
- **Exit combat:** Escape? → нет, бой завершается только исходом

## Этапы разработки
1. HTML + CSS + сетка + игрок + движение + стены поля
2. Масштабирование сетки по этажу + лестница + переход этажей
3. Монстры (один тип) + ИИ движения + бой (все 3 кнопки)

5. Предметы простые: монета, зелье
6. Уровни + XP
7. Снаряжение: слоты + пул + дроп из сундуков + экран сравнения

## Что НЕ делать
- Нет бэкенда / мультиплеера
- Нет canvas / WebGL — только CSS-grid + эмодзи
- Нет сетки больше 7×7
- Нет автобоя — все действия через меню
- Нет продажи снаряжения / крафта — только подбор и замена
