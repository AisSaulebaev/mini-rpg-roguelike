# IDLE RPG «Восхождение» — ПЛАН ВНЕДРЕНИЯ (stats-driven петля)

> Переводит `idle_rpg_design.md` в конкретный план кода. Числа помечены «(тюн)» —
> черновые, крутим в Ф5. Цель: уйти от фейкового боя («монстр умирает за N ударов,
> герой не получает урон») к реальной симуляции по характеристикам.

## Где мы сейчас
- Визуал готов: сцена, анимация боя (подход/файт), дроп-окно, счётчики глав, инвентарь-панель, бюст героя.
- Прогрессия глав работает (враг k/10 → глава → локация, золото/ядра/xp, элита/босс).
- Бой **фейковый**: фикс. число ударов, герой урона не получает, статы — декорация.

## 1. Модель данных (единый стейт → localStorage)

### 1.0 ДВА уровня — аккаунт (потолок) и герой (за золото)
- **Уровень АККАУНТА** (верхняя аватарка) растёт **от убийств** (xp с килла). Это **ПОТОЛОК**.
- **Уровень ГЕРОЯ** (вкладка Герои) качается **за золото** (`heroUpgradeCost`), герой становится сильнее, но **не выше уровня аккаунта**.
- Смысл: убийства поднимают потолок → золото подтягивает героев к нему = сток золота + «стена».

### 1.1 Hero — самостоятельная единица (свой level/gear/stats)
Каждый герой хранит своё: `level` (за золото, ≤ уровень аккаунта), шмот, статы. В команде до 3.
```js
{
  id:'warrior', name:'Воин', icon:'…hero_warrior.png',
  role:'tank',                 // профиль: tank | crit | nuker
  level:1,                     // качается ЗА ЗОЛОТО, кап = уровень аккаунта (xp у аккаунта, не у героя)
  gear:[null×8],               // оружие, броня, шлем, перчатки, ботинки, амулет, кольцо, питомец
  hp:100,                      // ТЕКУЩЕЕ hp в бою (для отхила/смерти)
  stats:{ maxHp,dmg,atkSpd,armor,crit,critX,regen,lifesteal,dodge,luck,pen }  // computed
}
```
**Прокачка уровнем** качает только **maxHp / dmg / armor** (формула уровня). Крит, скор.атаки,
вампиризм, уклонение уровнем НЕ растут — они **со шмота**. Уровень = «толще и больнее»,
билд = экипировка.

**Герои РАЗНЫЕ** (не клоны-с-большими-числами). Профиль = базовые коэффициенты + фишка:

| Герой | Роль | HP | Урон | Скор.атаки | Крит | Фишка |
|---|---|---|---|---|---|---|
| Воин | tank | ×1.35 | ×0.85 | 0.9/с | 5% | меньше входящего урона |
| Эльф | crit | ×0.8 | ×0.9 | 1.4/с | 12% | частые криты |
| Дракон | nuker | ×0.9 | ×1.5 | 0.7/с | 5% | редкий тяжёлый удар |
(коэффициенты — тюн)

Открытие нового героя = он с 1 уровня (свежий рост). Каждый качается отдельно.

### 1.2 Item (предмет)
```js
{ slot:0..7, rarity:'rare', ilvl:7,            // ilvl = глава дропа
  affixes:[{stat:'dmg',val:18},{stat:'crit',val:3}],
  icon:'🗡️', name:'…', sell:120 }
```
`power ∝ 1.12^ilvl`, распределяется по аффиксам. Кол-во аффиксов растёт с редкостью.

### 1.3 Enemy (не сохраняется — генерится из chapter/enemyIdx)
```js
{ hp,maxHp,dmg,atkSpd,atkCd, kind:'normal'|'elite'|'boss' }
```

### 1.4 Progress / Meta (частично уже есть)
```js
{ chapter, enemyIdx(1..10), location, gold, cores, activeHeroes[] }
```

## 2. Формулы (конкретные, тюн)

### 2.1 База героя по уровню (× коэффициент профиля из §1.1)
```
baseHP(l)    = 100 × 1.11^(l-1)     // l1=100, l10≈256, l30≈1730
baseDMG(l)   = 12  × 1.10^(l-1)     // l1=12,  l10≈28,  l30≈190
baseArmor(l) = 3 + l
atkSpd       = профиль (см. таблицу) + шмот
crit=профиль, critX=1.8, regen=1%maxHp/с, dodge=3%
```

### 2.2 Итог = база×профиль + Σ аффиксов шмота
Пересчёт (`recomputeStats(hero)`) при: level-up, надеть/снять предмет, открытие героя.

### 2.3 Удар героя → враг
```
crit? = rand()<stats.crit;  raw = stats.dmg × (crit?critX:1) × rand(0.9..1.1)
enemy.hp -= raw            // у врагов брони нет → полный урон
lifesteal>0: hero.hp += raw×lifesteal (кап maxHp)
```

### 2.4 Удар врага → герой
```
rand()<hero.dodge → промах («Уворот»)
DR = armor/(armor+100)   (кап 75%)
hero.hp -= enemy.dmg × (1-DR)
```

### 2.5 Враг (уже в коде — §3 дизайна)
```
enemyHP(c,k)  = 10 × 1.35^(c-1) × (1+0.05(k-1))   [×3 элита /×8 босс]
enemyDMG(c,k) = 2  × 1.28^(c-1) × (1+0.04(k-1))   [×1.5 /×2.5]
enemyAtkSpd   = 0.8/с
```

### 2.6 Шмот при дропе
```
ilvl=chapter; power=round(4×1.12^ilvl)×rarityMult   // 1/1.6/2.6/4.2/7
affixCount={common1,rare2,epic3,legendary4}; power → рандом по N аффиксам слота
sell=sellValue(chapter)
```

### 2.7 XP / уровень (уже есть)
```
xpFromKill(c)=6+3c (+25 элита,+80 босс);  xpNeed(l)=30×1.18^(l-1)
```

### 2.8 Экономика (сток золота)
```
heroUpgradeCost(l)=25×1.15^(l-1)
кап: hero.level ≤ accountLevel   // выше потолка не апнуть, пока не поднимешь аккаунт киллами
```
Продажа → золото → апгрейд героев (цена растёт быстрее дохода) → «стена» → открыть нового героя / фармить шмот.

## 3. Тик-движок боя (сердце Ф2)
Существующий `setInterval(tick,32)` (DT=0.032) гоняет деревья — на нём же крутим бой (работает и под virtual-time). PHASE 2 вместо fixed-hits — реальная симуляция по кулдаунам.

**Каждый тик, пока PHASE 2:**
```
за каждого живого героя:
  atkCd -= DT
  if atkCd<=0: анимация удара; raw→enemy.hp; lifesteal; atkCd=1/atkSpd
  hp += regen×DT (кап maxHp)
enemy.atkCd -= DT
if enemy.atkCd<=0 && живые: выбор цели; dodge? иначе monsterAttack→hp; atkCd=1/atkSpd
обновить HP-бары (враг+герои)
if enemy.hp<=0  → ПОБЕДА: fade, дроп, defeated(), следующий раунд
if все hp<=0    → ПОРАЖЕНИЕ (§3.1)
```
Существующие анимации (weaponStrike/heroStrike/monsterAttack) вешаем на события сима.

### 3.1 Смерть команды (−1 глава)
```
running=false → вспышка «ПОРАЖЕНИЕ»
chapter=max(1,chapter-1); enemyIdx=1; все hp=maxHp
renderProgress(); round()   // снова с подхода — не game-over, а откат
```

## 4. Экипировка (Ф3)
1. Дроп: шанс `8%+luck` (в тесте 100%), генерим item (§2.6).
2. Окно дропа: статы + **сравнение** с надетым (Δ +/− зелёным/красным).
3. Надеть: `gear[slot]=item` → `recomputeStats` → HP/статы обновились.
4. Продать: `addGold(item.sell)`.
5. (Позже) авто-продажа по правилу редкости.

## 5. Прокачка / экономика (Ф4)
- Пассивный level-up от боя (xp) → +статы.
- Ручной апгрейд за золото (кнопка) — основной сток.
- «Стена»: доход линеен, цена апа экспонента → открыть нового героя (lvl1, свежий рост).
- Ядра: с боссов (+2) → Магазин (сундуки/герои).

## 7. Фазы (конкретные задачи)
- **Ф1 — Стат-ядро:** `HEROES[]` c профилями, `recomputeStats`; статы-модалка и HP-числа = реальные. Поведение не меняется.
- **Ф2 — Реальный бой ← приоритет:** тик-симуляция (§3), обоюдный урон, крит/уворот/вампиризм/реген, смерть в обе стороны, −1 глава.
- **Ф3 — Шмот со статами:** генератор + аффиксы, окно сравнения, надеть=пересчёт, продать.
- **Ф4 — Уровни/экономика:** апгрейд за золото, стена, открытие героя = lvl1.
- **Ф5 — Сейв + баланс:** localStorage, тюнинг «(тюн)» чисел, шанс дропа, оффлайн-доход.

**Старт:** Ф1+Ф2 одним заходом (бой не заведётся без статов), косметику замораживаем.
